JavaScript basics

This page will cover some of the most essential JavaScript basics that we have learnt so far

Typing

JavaScript is loosly typed and is a dynamic language. The types of variables are therefore not immediately paired with a specific date type, and can be assigned and re-assigned new values on the spot. Example:

                        let money = 100; // type is a number
                        money = 'a lot of money'; // money is now a string
                        console.log(money);

                        Output: a lot of money

Declarations

In JavaScript we declare variables with three different keywords:

var declares a variable

let declares a block-scoped variable

const declares a block-scoped, read-only constant

For our work we never use var as global variables can lead to a number of unintended bugs. Most prominently a failure to account for JavaScript hoisting.

Furthermore we have no need to use var as all of our variable creating needs are covered using let and const

To read more about why we don't use var, please read: Why You Shouldn’t Use ‘var’ Anymore.

Type coercion

JavaScript uses something called type coercion when two variables of different types are compared. One of them will be converted into an "equivalent" of the other variables type

This all happens implicitly! This can therefore lead to a number of bugs if you are not careful. Example:

const valueString = '100';

const valueNumber = 5;

let sum = valueString + valueNumber;

console.log(sum);

 Output: 1005

A way to get around type coercion when comparing values is by using the === operator instead of the == operator when comparing values.

Using === will not use any type coercion and will return true if, and only if, both values and types are identical for the two variables being compared.

Callback functions

We have previously learned about regular JavaScript functions, but for this semester we have used callback funcitons a whole lot.

Basically a callback function is a function that takes another function as an argument, which is then invoked inside the outer function. This sounds complicated but it's actually not.

Example:

                        function print(callback) {
                            callback();
                        }

In this example the print function takes another function as a parameter and calls it inside of the function.

Using callback functions we are able to make the code not run sequentially. We are therefore able to now develop asynchronous JavaScript

Alternatively we can define a function direcly inside another function. This is called an anonymous function: 

                        setTimeout(function() {  
                            console.log("This message is shown after 3 seconds");
                        }, 3000);

There is another way to define an anonymous function and that is by declaring it as an arrow function. Arrows functions is something that we have preferred to use during the semester, but using the above syntax is also perfectly fine - it just comes down to preference: 

                        setTimeout(() => { 
                            console.log("This message is shown after 3 seconds");
                        }, 3000);
