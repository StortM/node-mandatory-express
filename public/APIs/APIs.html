RESTful APIs

A RESTful API is an architectural style for an API that uses HTTP requests to access and use data. That data can be used to GET, PUT, POST and DELETE data types, which refers to the reading, updating, creating and deleting of operations concerning resources.

RESTful API design

In general a RESTful API uses the following endpoints. The below table can be used as a template as to how to desiign your own RESTful API:

HTTP Method Endpoint Description
GET /students Retrieve all students
GET /students/id Retrieve student by id
POST /students Create user(s)
PUT/PATCH /students/id Update single student
DELETE /students/id Delete student by id

The following endpoints are rarely used and should be implemented with caution. Although some of them might have niche use cases.

HTTP Method Endpoint Description Why this is rarely used
POST /students/id Create user at specific id This is very dangerous! The user should never be able to create users at specific ids. This should handled properly by the databse implementation.
PUT/PATCH /students Update all users Very specific case where all students should be updated in the way.

PUT vs PATCH

Notice how we use PUT/PATCH interchangeably above? This is because they in a large way represent the same functionality (updating a resource), although there are some differences. The key difference is that the PUT method replaces the entire resource with a new one, whereas PATCH has the ability to update only a part of the resource.

During our semester we have preferred using PATCH as it is seen as more efficient and specific to the general use case.
