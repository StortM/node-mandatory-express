Node

Node.js is JavaScript runtime environment that we have used to create web website and APIs. It enables us to run JavaScript server-side and not only inside the browser.

How to run files

Using the NodeJS it is easy breezy to run node projects. If you have NodeJS installed you can simple type node *filename.js* into your terminal and your project will run

package.json

Similar to how we used to have a pom.xml file in Java, Node uses a similar structure utilizing the file package.json which works quite similar (but obviously it's in json format instead of xml)

Installation of dependencies from the package.json using npm is quite easy as well. You just type: npm install

Something to note is that even though you don't use any dependencies you will have to have a file in your root directory callled "package.json" with an empty json inside. This is required to run your node application. This will become quite obvious after you have created your first application and ran the npm install command, as you will have quite a big folder called node_modules in your root directory.

As is with the pom.xml file in Java, the package.json contains more than just dependencies. It can also contain metadata about your project as well as scripts!

An example of a package.json file:

                    {
                        "scripts": {
                          "start": "node app.js",
                          "start-dev": "cross-env PORT=3000 nodemon app.js"
                        },
                        "dependencies": {
                          "express": "^4.17.1"
                        },
                        "devDependencies": {
                          "cross-env": "^7.0.3"
                        }
                      }

In this example we have a dependency to express as well as a few scripts for the developer to start the project in production mode or developer mode

Another hugely important feature of the package.json file is that we can seperate our dependencies that are needed for production and development. In the above example this is done using the json objects "dependencies" and "devDependencies"
